# 四大新功能呈現方式發想

基於現有網站的設計語言（ASCII art 開場、blur-reveal 動態、暖沙色卡片、glassmorphism、WebGL 背景），以下是每個功能的 5 種創意呈現方式。

---

## 1. 🎵 音樂 (Background Music)

| #           | 方案名稱                                | 描述                                                                                                                                                                                                                    |
| ----------- | --------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A** | **脈搏律動條 (Pulse Bar)**        | 固定在畫面底部的超薄玻璃條，像心電圖一樣隨音樂節奏脈動。點擊可展開成迷你唱片機 UI（唱片旋轉 + 歌名）。未播放時只是一條安靜的光線，播放中會隨 `AnalyserNode` 頻率數據產生 `波形微動畫`。                             |
| **B** | **漂浮音符粒子 (Floating Notes)** | 音樂播放時，從畫面底部緩慢升起半透明的音符粒子（♪ ♫），融入 WebGL LivingBackground 的色彩。控制按鈕是一個固定在右下角的小圓形按鈕，hover 時展開歌名。靜音時粒子消散。                                                 |
| **C** | **黑膠唱片角落 (Vinyl Corner)**   | 畫面右下角露出半個黑膠唱片邊緣，滑鼠靠近時唱片滑出並旋轉。唱片上印著「Yuan & 4J's Playlist」。點擊唱片切換播放/暫停，唱片旋轉速度跟著音樂節奏。整個元素用 `glassmorphism` 風格的背景模糊。                            |
| **D** | **呼吸光暈 (Breathing Aura)**     | 完全不顯示任何 UI 控制項。音樂在用戶點擊 ASCII Portal 的「請點擊」後自動播放。整個網站的 LivingBackground 會隨音樂振幅微微 `脈動亮度`。角落有一個極小的靜音按鈕（hover 時才顯現），按下後光暈停止、音樂淡出。         |
| **E** | **ASCII 波形 (ASCII Waveform)**   | 延續 ASCII Portal 的美學，在 header 或 footer 區域顯示一行用 ASCII 字元（如 `▁▂▃▄▅▆▇█`）組成的即時音頻波形。點擊波形切換播放狀態。波形的字元密度和高度隨音樂頻率變化，靜音時退化成一條安靜的 `▁▁▁▁▁`。 |

> [!TIP]
> **我的推薦**：方案 **A（脈搏律動條）** 或 **D（呼吸光暈）**。A 提供清楚的控制又不搶版面；D 最沉浸但需要考慮 autoplay 政策（手機瀏覽器會擋）。

> 💡 **參考實作**（依 21st.dev inspiration + ui-ux-pro-max 準則校準）
>
> 採 **方案 A**。21st 的真實前例（Music Toggle btn）揭露一個關鍵取捨：
> - **不要用 `AnalyserNode`**：features.md 原本寫接 Web Audio 的 `AnalyserNode` 取頻率，但真實前例是用 `setInterval` 每 100ms 給 N 根 bar 隨機高度（`Math.random()*0.8+0.2`）「假裝」波形。婚禮律動條純裝飾，假波形視覺上完全一樣，實作量約 1/10，且免去解碼音訊、接 audio graph、手機耗電的問題。
> - **bar 高度建議用 spring 緩動**：前例用 framer-motion spring（stiffness 300, damping 10）。本專案沒裝 framer-motion → 改用 CSS `transition: height` 或既有 GSAP，效果等同。
> - **常駐底部、tap 觸發**：控制必須是常駐可點的實體元素（符合 ui-ux-pro-max `Hover vs Tap` High — 手機沒有 hover）。靜音時 bar 退化為一條安靜的低平線（`Array(N).fill(0.1)`）。
> - **autoplay 政策**：預設靜音、第一次使用者互動（點 ASCII Portal「請點擊」）才解鎖 + 播放，避免賓客在辦公室開網站突然出聲。
> - **動畫節制**：只動「這一條」bar（符合 `Excessive Motion` High：每 view 最多動 1–2 元素），不要再讓 LivingBackground 跟著脈動（這是方案 D 的坑）。

## 2. ⏳ 日期倒數 (Countdown to Oct 3, 2026)

| #           | 方案名稱                                    | 描述                                                                                                                                                                                                                                  |
| ----------- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A** | **沙漏翻轉 (Hourglass Flip)**         | 用 ASCII art 畫一個沙漏，沙粒（用 `.` 和 `:` 字元）持續從上半部掉到下半部。沙漏旁邊用超大字體顯示天數，每秒翻動一個數字（flip-clock 效果）。放在「那天見」finale section 裡取代目前靜態的 `2026.10.3`。                         |
| **B** | **時間碎片拼圖 (Time Mosaic)**        | 把倒數的「天 / 時 / 分 / 秒」做成四張 `glassmorphism` 卡片，每張卡片內的數字用 `blur → 清晰` 的轉場做翻牌動畫。卡片排列方式跟現有的 `schedule-scenes` 三格佈局一致，但改為四格。底下一行小字：「距離那天，還有這麼多個心跳。」 |
| **C** | **軌道倒數 (Orbital Timer)**          | 一個圓形軌道，上面有一個小光點代表「今天」，沿軌道移動到終點「10.3」。軌道用細線畫成，進度用漸層填充。圓心顯示剩餘天數的大字。整體風格像天文儀器，呼應「優聖美地」的戶外自然感。                                                      |
| **D** | **打字機倒數 (Typewriter Countdown)** | 用打字機效果逐字打出一段話：「距離 Yuan & 4J 的婚禮，還有 ███ 天 ██ 小時 ██ 分 ██ 秒」，數字部分持續即時更新。字元用等寬字型，搭配閃爍的游標 `▌`。風格跟 ASCII Portal 完美銜接。                                          |
| **E** | **花瓣飄落計數 (Petal Rain Counter)** | 在 finale section 的背景中，每秒飄落一片花瓣（CSS 動畫）。畫面中央用巨大的數字顯示剩餘天數，數字本身用花瓣紋理填充（CSS `background-clip: text`）。當倒數歸零時，花瓣變成煙火爆炸效果。                                             |

> [!TIP]
> **我的推薦**：方案 **B（時間碎片拼圖）** 或 **D（打字機倒數）**。B 跟現有卡片設計語言一致；D 跟 ASCII Portal 的打字美學完美呼應，而且實作相對簡單。

> 💡 **參考實作**（依 21st.dev inspiration + ui-ux-pro-max 準則校準）
>
> 21st 回了三個真實倒數元件（ShiftingCountdown / AnimatedNumberCountdown / Ark Timer），三者揭露的共通實作要點：
> - **數字必須固定寬度防 CLS**：三個前例**全部**用 `min-w-[2ch]` 或 `minimumIntegerDigits: 2`（補零）固定數字寬度。這正中 ui-ux-pro-max 的 `Content Jumping`（High）—— 倒數每秒更新，若用比例字型，`1`→`8` 寬度不同會每秒抖動版面。**方案 B 必須用等寬字 + 固定寬容器**；方案 D 用等寬字天然免疫。
> - **翻動動畫抓 0.35s**：ShiftingCountdown 的數字「向上滑出→新數字滑入」用 `duration: 0.35`，貼合 ui-ux-pro-max 的 `Duration Timing`（micro-interaction 150–300ms 上緣）。方案 B 的 blur→清晰翻牌也壓在此區間，別超過 500ms。
> - **數字滾動效果可加分**：AnimatedNumberCountdown 用 `@number-flow/react` 做「里程表滾動」翻數字，比單純翻牌精緻。可考慮把這個概念（用 CSS/GSAP 自製，不引入該套件）用在方案 D 的數字部分。
> - **計時邏輯**：用單一 `setInterval(fn, 1000)`，`distance = end - now`，再除 DAY/HOUR/MINUTE/SECOND 拆解；卸載時 `clearInterval`。注意目標時區（婚禮 2026-10-03 為台北時間，建議用 `+08:00` 明確標註，與 `src/lib/rsvp.ts` 既有的 deadline 寫法一致）。
> - **歸零狀態別漏**：features.md 只在方案 E 提了煙火。B/D 也要設計倒數歸零文案（如「今天，就是今天 ♡」），否則婚禮當天顯示「還有 0 天」很怪。
> - **放置位置**：finale section，取代或增強現有靜態的 `2026.10.3`。

## 3. 👔 Dress Code

| #           | 方案名稱                                        | 描述                                                                                                                                                                                              |
| ----------- | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A** | **色票展示牆 (Color Swatch Wall)**        | 用一排可互動的色票卡片，每張卡片是一個建議色系（如「霧藍」「暖沙」「淺玫瑰」）。Hover 時卡片翻轉，背面顯示穿搭建議文字。色票的顏色跟網站主色調一致，讓賓客直覺理解婚禮的色彩氛圍。                |
| **B** | **衣架展示區 (Wardrobe Rail)**            | 用 CSS 畫一根橫向衣架桿，上面掛著幾個衣架圖示（男女各一組）。每個衣架下方用極簡線條畫出衣服輪廓，hover 時展開詳細說明（如「男士：西裝外套 + 深色長褲」）。整個元素可以左右滑動瀏覽。              |
| **C** | **Mood Board 拼貼 (Collage Board)**       | 用生成的圖片做成一個 Pinterest 風格的 mood board，展示婚禮的穿著氛圍（戶外、半正式、自然色系）。圖片用 `masonry layout` 排列，帶有 `glassmorphism` 標籤浮在圖片上標示「✓ 推薦」「✗ 避免」。 |
| **D** | **Do & Don't 對比翻卡 (Flip Comparison)** | 兩欄式佈局：左邊「推薦 ✓」右邊「避免 ✗」。每一項是一張可翻轉的卡片，正面是一個代表性 emoji 或圖示，翻轉後顯示文字說明。推薦欄用暖色調，避免欄用灰色調。帶有 `blur-reveal` 滾動動畫。          |
| **E** | **互動穿搭指南 (Style Guide Wizard)**     | 一個簡單的互動流程：先問「你是？」（男生/女生/不限），然後顯示對應的穿搭建議。每一步都有動畫轉場，最後一步顯示完整的 dress code 總結卡片，可以截圖保存。風格類似現有 RSVP 表單的 wizard 步驟。    |

> [!TIP]
> **我的推薦**：方案 **A（色票展示牆）** 或 **D（Do & Don't 翻卡）**。A 視覺衝擊力最強且跟網站色彩系統完美融合；D 資訊最清楚，賓客一目了然。

> 💡 **參考實作**（依 21st.dev inspiration + ui-ux-pro-max 準則校準）
>
> 採 **方案 A**，但**移除 hover 翻轉**。21st 回的 ColorPaletteCard 揭露一個輕巧互動：hover 該色塊時 `flex: 2` 展開放大並顯示色碼。但對照 ui-ux-pro-max 準則，這方案踩了三個必修坑：
> - 🔴 **hover 在手機無效**（`Hover vs Tap` High）：婚禮網站九成賓客用手機，翻轉/hover 展開他們碰不到。→ 改成 **tap 展開**，或乾脆**文字常顯**（最穩）。
> - 🔴 **不能只靠顏色**（`Color Only` High）：色票對色盲賓客（男性約 8%）資訊量大減。→ 每個色票**一定要有色名文字**（如「霧藍 Dusty Blue」），不只色塊。可借 21st BrandingCard 給每個色塊加 `aria-label="霧藍 #aabbcc"` 的做法補無障礙。
> - 🔴 **對比 4.5:1**（`Color Contrast` High）：你的色系「霧藍/暖沙/淺玫瑰」偏淺，色塊上若壓白字會對比不足。→ 文字用**深色**（呼應網站 `--friends-ink` 或 `#2f302f`），不要白字。
> - **建議結構**：一排 `flex` 色塊，每塊內含「色塊背景 + 深色色名文字 + 一句穿搭情境（如『主色調，建議外套或洋裝』）」，全部常顯、無互動門檻；想加趣味就用 tap 微放大（非 hover、非翻轉）。
> - **放置位置**：流程 section 後面，新增獨立 section。

## 4. 🚌 接駁車時間 (Shuttle Bus Schedule)

| #           | 方案名稱                                    | 描述                                                                                                                                                                                                                  |
| ----------- | ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A** | **時間軸路線圖 (Timeline Route)**     | 用一條垂直的虛線路線，沿途標記每個發車時間點和地點（如「14:00 台北車站出發 → 15:30 抵達優聖美地」）。路線上有一個小巴士圖示 🚌 用 CSS 動畫沿路線移動。搭配 `scroll-driven animation`，滾動時巴士跟著前進。         |
| **B** | **車票式卡片 (Ticket Cards)**         | 把每個班次做成一張復古車票樣式的卡片（左邊出發地、右邊目的地、中間虛線撕裂線、底部時間）。卡片用暖沙色紙張質感，邊緣有鋸齒裁切效果。去程和回程用不同色系區分。Hover 時卡片微微浮起 + 投影加深。                       |
| **C** | **站牌互動板 (Bus Stop Board)**       | 模擬真實公車站牌的 LED 顯示板效果。用等寬字型 + 綠色/琥珀色字在深色背景上顯示班次資訊，帶有掃描線效果。每個班次像 LED 跑馬燈一樣輪播。底下有「加入行事曆」按鈕。                                                      |
| **D** | **雙向箭頭時刻表 (Arrow Timetable)**  | 畫面分上下兩區：上方「去程 →」下方「← 回程」。每個班次是一個帶箭頭的 `glassmorphism` 膠囊標籤，排列在對應方向的時間軸上。箭頭方向暗示行進方向。出發地和目的地用固定在兩端的大標題標示。                           |
| **E** | **地圖路徑動畫 (Map Path Animation)** | 在現有 Google Maps 區域旁邊（或整合在 venue section 內），加一個簡化的路徑動畫。用 SVG 畫出從上車點到優聖美地的抽象路線，一個小圓點沿路線移動。路線旁標注各班次時間。點擊時間可展開詳細資訊（上車地點、預估車程等）。 |

> [!TIP]
> **我的推薦**：方案 **B（車票式卡片）** 或 **A（時間軸路線圖）**。B 跟現有的 `schedule-scenes` 卡片設計一致，又有趣味性；A 的滾動驅動動畫最沉浸但實作複雜度較高。

> 💡 **參考實作**（依 21st.dev inspiration + ui-ux-pro-max 準則校準）
>
> 採 **方案 B**，但**去擬物化**。21st 回了三個機票/登機證元件，其中 FlightTicket 幾乎就是「車票卡片」的現成骨架，可直接參考其版面結構：
> - **車票感用 CSS 虛線即可，不必堆紋理**：FlightTicket 的撕裂線是純 CSS `border-t-2 border-dashed`、存根條碼是 SVG `<rect>` 畫的——**完全不靠紙張紋理/噪點/鋸齒**。這正好避開 ui-ux-pro-max 對 skeuomorphism 的警告（`Performance: ❌ Poor`、`Accessibility: ⚠ 紋理降低可讀性`）。features.md 原本寫的「紙張質感 + 鋸齒裁切」其實不必做那麼重，**虛線 + 存根區就有車票感**。
> - **版面結構**：左「出發地」右「目的地」雙欄 + 中間虛線/圖示/車程（如「台北 →（車程約 1.5h）→ 優聖美地」）+ 底部存根區放時間。借 FlightCard 的「中間虛線 + 圖示 + duration」表達行進。
> - **套既有風格**：前例全是 framer-motion + shadcn `Card`/`Badge`，一行都不能直接貼。用本專案既有的暖沙玻璃卡片 class（呼應 `schedule-scenes`）重做。
> - **去程/回程區分**：用不同邊框色或標籤（非僅顏色，加文字「去程／回程」，符合 `Color Only` 準則）。
> - **功能性優先**：接駁車是「賓客快速查幾點、哪上車」的查詢資訊，掃讀體驗 > 炫動畫。方案 A 的 scroll-driven 巴士移動若要做，務必在 `prefers-reduced-motion` 時退化成靜態列表（本專案 `FriendsMotionController` 已有 matchMedia 判斷可參考）。
> - **放置位置**：地點 section 後面（跟交通資訊放一起）。可加「加入行事曆」按鈕（21st LED 板前例有此模式）。

## 💡 inspiration 跑完的共通洞見（四功能橫向）

> [!NOTE]
> 用 `21st.dev` 抓真實前例 + `ui-ux-pro-max` 準則校準後，四個功能的三個共通結論：
>
> 1. **手機優先、tap-first**：原方案有大量 hover 互動（黑膠 hover 滑出、色票 hover 翻轉、衣架 hover 展開）。婚禮網站九成賓客用手機，**hover 等於碰不到**。所有重要互動改 tap 或常顯。
> 2. **借結構、不借 code**：21st 回的元件全是 Tailwind + framer-motion + shadcn，本專案是手寫 CSS + GSAP/Lenis/three.js——**程式碼一行都不能直接貼**（會引入新依賴或風格打架）。價值在「版面結構與互動模式參考」，用既有風格重做。
> 3. **減重勝過堆料**：真實前例常用最輕的手法達成效果（假波形用 `setInterval` 隨機高度、車票用 CSS 虛線、倒數用 `min-w` 防抖）。別過度追求 AnalyserNode / 紙張紋理 / 複雜 scroll 動畫，那些正是 ui-ux-pro-max 標記的效能/可讀性坑。

### 校準後的推薦組合（手機優先 + 過準則）

| 功能 | 推薦方案 | 一句話理由 |
|---|---|---|
| 🎵 音樂 | **A 脈搏律動條（假波形）** | 常駐可點、只動一元素、不靠 autoplay/hover/AnalyserNode |
| ⏳ 倒數 | **D 打字機**（或 B 時間碎片，需固定數字寬） | 等寬字免 CLS、同 ASCII 美學、最易實作 |
| 👔 Dress Code | **A 色票牆（文字常顯，不靠翻轉）** | 修掉 color-only + hover + 對比三坑 |
| 🚌 接駁車 | **B 車票卡片（去擬物，CSS 虛線+存根）** | 保留車票隱喻、避開 skeuomorphism 效能坑 |
